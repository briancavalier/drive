## Problem statement
Factory planning run `23724560686` failed during the budget preflight before Codex execution. The guardrail script treated `plan.md` as a required input even for `plan` mode, but that artifact does not exist until the planning stage has actually run.

## Motivation
Planning runs should be able to estimate cost and proceed to Codex execution without depending on artifacts that are only produced later in the same stage. As written, the preflight turned a normal planning run into a configuration failure and blocked issue-driven factory progress.

## What changed
- Short-circuit non-implement stages in `scripts/enforce-stage-budget.mjs` before reading `plan.md`.
- Preserve the existing implement-stage behavior, including failing if the plan artifact is missing when it is actually required.
- Add a regression test covering `plan` mode before `plan.md` exists.

## Reviewer context
- The failing run was a `plan / run` job for issue `#109` on branch `factory/109-add-repair-exhaustion-decision-interventions`.
- The failure occurred immediately after `Estimate stage cost` and inside `node scripts/enforce-stage-budget.mjs`.
- Focus review on whether non-implement stages now avoid false configuration failures while implement-stage budget enforcement still reads the plan artifact when needed.
